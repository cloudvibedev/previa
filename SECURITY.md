# Security Policy

## Reporting a Vulnerability

If you believe you found a security issue in Previa, please do not open a
public GitHub issue with exploit details.

Instead, report it privately to:

- `previa@cloudvibe.dev`

Please include:

- a clear description of the issue
- affected components or routes
- reproduction steps or a proof of concept
- the potential impact
- any suggested mitigation, if available

We will review the report and follow up as quickly as possible.

## Scope

Security reports are especially helpful for issues involving:

- authentication and authorization
- runner communication and `RUNNER_AUTH_KEY`
- exposed APIs or MCP endpoints
- proxying behavior
- unsafe defaults that could affect deployed environments

## Public Disclosure

Please give us reasonable time to investigate and prepare a fix before public
disclosure.
